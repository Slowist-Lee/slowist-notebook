    ![[Pasted image 20240724144444.png]] 在MySQL中，GROUP_CONCAT() 是一个聚合函数，用于将分组中的多个值连接成一个字符串。它常用于在分组查询中，将每个组内的值连接起来并以逗号或其他分隔符分隔。下面是详细的解释和一些使用示例。

    基本语法¶

    SQL
    1
    GROUP_CONCAT([DISTINCT] column_name [ORDER BY clause] [SEPARATOR 'separator'])
    - column_name: 要连接的列。 - DISTINCT: 可选，表示只连接唯一的值。 - ORDER BY clause: 可选，指定连接的顺序。 - SEPARATOR: 可选，指定分隔符，默认是逗号（,）。

    示例¶

    假设有一个表 employees，结构如下：

    id name department
    1 Alice HR
    2 Bob IT
    3 Charlie HR
    4 David IT
    5 Eve HR

    1. 将同一个部门的员工姓名连接起来¶

    SQL
    1
2
3
    SELECT department, GROUP_CONCAT(name) AS employees
FROM employees
GROUP BY department;
    结果：

    department employees
    HR Alice,Charlie,Eve
    IT Bob,David

    2. 使用 DISTINCT 只连接唯一值¶

    SQL
    1
2
3
    SELECT department, GROUP_CONCAT(DISTINCT name) AS employees
FROM employees
GROUP BY department;

    3. 指定连接顺序¶

    SQL
    1
2
3
    SELECT department, GROUP_CONCAT(name ORDER BY name ASC) AS employees
FROM employees
GROUP BY department;
    结果：

    department employees
    HR Alice,Charlie,Eve
    IT Bob,David

    4. 指定分隔符¶

    SQL
    1
2
3
    SELECT department, GROUP_CONCAT(name SEPARATOR '; ') AS employees
FROM employees
GROUP BY department;
    结果：

    department employees
    HR Alice; Charlie; Eve
    IT Bob; David

    注意事项¶

    • GROUP_CONCAT 函数有一个默认的最大长度（通常是1024个字符），可以通过设置 group_concat_max_len 系统变量来增加这个限制。
      SQL
      1
      SET SESSION group_concat_max_len = 10000;
    • 如果组内的连接结果超过了最大长度，结果会被截断。

    通过使用 GROUP_CONCAT()，可以方便地将分组后的多个值连接成一个字符串，从而简化了某些数据处理的需求。

    group_concat的缺陷：如果有None，就会返回None，并成一项了

    无列名注入：

    这段话描述了一种 SQL 注入攻击的技巧，主要用于在不知道表列名的情况下，通过自定义列名来获取表中的数据。具体步骤如下：

    1. 构造自定义列名的数据：
    2. 首先，构造一个包含多个列的查询。通常使用一些简单的数字作为列名，这里使用了 1, 2, ..., n。
    3. 这个部分的查询看起来像这样：
      SQL
      1
      SELECT 1, 2, ..., n

    4. 其中，n 表示列的数量。

    5. UNION 操作：

    6. 使用 UNION 操作符，将上一步的查询结果与目标表中的数据拼接在一起。
    7. 例如：
      SQL
      1
      UNION SELECT col_name1, ..., col_nameN FROM table_name

    8. 这里的 col_name1, ..., col_nameN 是目标表中的实际列名。

    9. 获取数据：

    10. 将上面构造的查询作为一个子查询，并使用 GROUP_CONCAT 函数来聚合数据。
    11. 完整的查询语句如下：
      SQL
      1
      SELECT GROUP_CONCAT(`k`) FROM (SELECT 1, 2, ..., n UNION SELECT col_name1, ..., col_nameN FROM table_name) custom_name

    具体解释¶

    假设我们有一个目标表 users，其列名和数据如下：

    Text Only
    1
2
3
4
    | id | username | password |
|----|----------|----------|
| 1  | alice    | 12345    |
| 2  | bob      | 67890    |

    如果攻击者不知道 users 表的列名，他们可以使用如下方法来进行 SQL 注入：

    1. 构造查询：

      SQL
      1
      SELECT 1, 2, 3

    2. 使用 UNION 拼接目标表的数据：

      SQL
      1
      UNION SELECT id, username, password FROM users

    3. 完整的注入语句：

      SQL
      1
      SELECT GROUP_CONCAT(`k`) FROM (SELECT 1, 2, 3 UNION SELECT id, username, password FROM users) custom_name

    4. 解释：

    5. 第一个 SELECT 1, 2, 3 创建了一个具有三个列的查询结果。
    6. UNION SELECT id, username, password FROM users 将 users 表中的数据拼接到第一个查询结果的后面。

    7. GROUP_CONCAT('k') 函数用于将所有结果拼接成一个字符串。

    8. 执行结果：

    9. 假设我们将 k 替换为实际的列名（例如 username），查询语句变为：
      SQL
      1
      SELECT GROUP_CONCAT(username) FROM (SELECT 1 AS username, 2, 3 UNION SELECT id, username, password FROM users) custom_name
    10. 这将返回用户表中的所有用户名拼接成的一个字符串，例如 "alice,bob"。

    通过这种方式，即使攻击者不知道表的列名，也可以通过 SQL 注入获取表中的数据。
