    2.1 通用寄存器¶

    • 所有寄存器都是16位的，可以放两个字节
    • AX BX CX DX
    • 可以分成两个独立的8个字节的寄存器使用
      • e.g. AX \(\rightarrow\) AH & AL;
      • AH：高8位，AL：低8位
      • 既可以看成一个数据的高8位和低8位，也可以看成两个独立数据
    • 汇编指令：
      • mov ax,18 → AX=18
      • mov ah,78 → AH=78
      • add ax,8 → AX=AX+8
      • mov ax,bx → AX=BX
      • add ax,bx → AX=AX+BX

    Warning

    指令的两个操作对象的位数应当是一致的！ 错误示范： - mov ax, bl x 8位和16位寄存器不可以传送数据 - mov bh, ax x 16位和8位寄存器不可以传送数据 - mov al, 20000 x 8位寄存器最大存放值为255的数据（20000是256...） - add al, 100H x 不能将高于8位的数据加到8位寄存器里

    寄存器分类¶

    8086 内部一共有 14 个 16 位寄存器，8 个通用寄存器，4 个段寄存器，2 个控制寄存器

    • 通用寄存器
      • 数据寄存器：ax bx cx dx，用于存放数据。每个寄存器为 16 位，可分为高低 8 位，高 8 位分别为 ah bh ch dh，低八位分别为 al bl cl dl，且如果只修改八位，则不会影响到另外八位（比如溢出不会从 al 溢出到 ah）
        • ax：累加器，常用作累加，且在乘除法、串运算、IO 中有专门用处
        • bx：基址寄存器，常用来存放寻址的基址
        • cx：计数寄存器，常用在循环中作为计数器
        • dx：数据寄存器，存放 IO 端口地址，且在双字运算中作为 ax 的扩展高十六位
      • 变址寄存器：si di，常用来寻址
        • si：源变址寄存器
        • di：目的变址寄存器
      • 指针寄存器：sp bp，常与 ss 构成远指针
        • sp：堆栈指针寄存器，ss:sp 指向堆栈顶端
        • bp：基址指针寄存器，常用 ss:bp 指向堆栈中数据
    • 段寄存器：cs ss ds es
      • cs：代码段寄存器，存放代码段的段地址
      • ss：堆栈段寄存器，存放堆栈段的段地址
      • ds：数据段寄存器，存放数据段的段地址
      • es：附加数据段寄存器，存放附加数据段段地址
    • 控制寄存器：ip fl
      • ip：指令指针寄存器，存放代码段的偏移地址， cs:ip 指向下一条指令的首字节，不能直接访问 ip
      • fl：标志寄存器，16 位但只使用 9 位表示 6 个状态标志和 3 个控制标志（df if tf）
        • of：overflow flag 溢出标志，第 11 位，表示运算是否溢出
        • df：direction flag 方向标志，第 10 位，标志字符串处理指令中处理信息的方向
        • if：interrupt flag 中断标志，第 9 位，是否允许外部硬件中断
        • tf：trace flag 跟踪标志，第 8 位，CPU 是否进入单步工作状态
        • sf：sign flag 符号标志，第 7 位，运算结果的符号（1 表示负）
        • zf：zero flag 零标志，第 6 位，运算结果是否为 0
        • af：auxiliary carry flag 辅助进位标志，第 4 位，记录运算时第三位是否向左侧产生进借位
        • pf：parity flag 奇偶标志，第 2 位，结果操作数中低 8 位 1 的个数是否为偶数
        • cf：carry flag 进位标志，第 0 位，运算时最高位是否向左侧有进借位

    80386 32 位寄存器一共有 16 个

    • 通用寄存器
      • eax ebx ecx edx，可以使用例如 ax 表示低 16 位，ah al 分别表示低 16 位中的高低 8 位
      • esi edi，低 16 位为 si di
      • esp ebp，低 16 位为 sp bp
    • 段寄存器：cs ss ds es fs gs（多了两个附加数据段寄存器）
    • 控制寄存器：eip eflags

.death:
  hlt  ; "Halt and catch fire" - 停止执行代码。立即。结束。不再回来。一切都结束了。天空中的死亡激光打到你了。等等。

.next_test:  ; 需要注意的是，我使用的是NASM汇编器（这相当标准），它提供了很多不错的预处理功能，就像C++中的`#include`一样。
            ; NASM提供的功能之一是'本地标签'，所以任何前面带点（"."）的标签都被视为函数（前一个标签前面没有点）的本地标签，因此你可以在你的代码库中的较大标签（暂时称为函数）中有相同的标签，仅为方便起见。
  cmp cl, 0
  je .success  ; 如果cl确实为零，跳过死亡！

  .success:  ; 顺便说一下，空白被忽略了...就像忽略我一样

  mov ax, 0x0003

  mov al, 't'
  mov ah, 0x0e      ; <- 问题4
  int 0x10          ; 同样的新东西。可怕！

  ; `int` 是一个中断。退一步思考我们程序员编写的代码可以做什么，代码除了操作数据和做决策（分支）之外什么也做不了。
  ; 如果我们希望我们编写的代码有意义的影响，我们需要与环境互动 - 将我们在代码中操作的数据表现出来，使其对最终用户有用。
  ; 这意味着我们需要调用我们代码运行的系统的API。在这段代码的情况下，我们需要与主板的系统 - BIOS - 交互，并请求它代表我们与屏幕互动（以后可能还有其他东西，比如USB驱动器等）。
  ; 我们在这里使用的中断是中断0x10。你可以在[这里](https://en.wikipedia.org/wiki/INT_10H)阅读它的所有功能。但是，实际上，根据AH的值，这个中断可以做很多不同的事情。AH + 0x0e允许我们打印存储在寄存器a的低8位部分中的8位ASCII字母：al。

  ; 但这对我们来说有点没用。它只能打印一个字母。我们可以做得更好。

  ; 首先让我们定义一个要打印的字符串，但请记住...现在我们在代码中间定义垃圾数据，所以我们需要跳过它以便没有尝试解码我们的文本字符串
  mov ax, .string_to_print
  jmp print_string
  .string_to_print: db "acOS", 0x0a, 0x0d, "  by Elyk", 0x00  ; 标签：<元素大小> <元素数组>
  ; db代表定义字节，有db, dw, dd, dq, dt, do, dy, 和 dz。我刚刚学到其中的三个存在。这并不是特定于汇编的知识。没关系。https://www.nasm.us/doc/nasmdoc3.html
  ; 数组可以是"字符串"或逗号分隔的值。

; 现在让我们做一个完整的'函数'来打印字符串
print_string:
  .init:
    mov si, ax  ; 我们没有传递参数的语法方法，所以我只是通过ax传递函数的第一个参数 - 要打印的字符串。

  .print_char_loop:
    cmp byte [si], 0  
; 括号内的表达式被解释为"地址"表达式中的任何内容.. 这与C语言中的解引用操作符完全相同

; 因此，在这种情况下，si是一个指针（它是从ax复制的指针（第183行），它是这个"函数"的第一个"参数"，它是我们试图打印的字符串的指针）
                      ; 如果我们当前指向一个空字节，我们就到了字符串的结尾...使用空终止字符串（第178行的字符串定义末尾的零）
    je .end

    mov al, [si]  ; 因为这被视为si的解引用，我们获取的是si处的字节...`al = *si`

    mov ah, 0x0e  ; <- 问题5!
    int 0x10      ; 实际打印字符

    inc si        ; 增加指针，转到下一个字符
    jmp .print_char_loop
    .end:

print_string_right:
  ; 正确打印字符串

  jmp .past_data  ; 提醒：本地定义标签，还记得：跳过下面的垃圾数据
  .important_string_to_print: db "lol kek ", 0
  .past_data:

  mov bp, .important_string_to_print  ; 字符串指针

  mov cx, 0x0007   ; 字符串长度
  mov bx, 0000000000001111b  ; 黑底白字

  mov ax, 0x1301  ; 这个打印整个字符串！（还清除了所有位，只剩低位...颜色模式）
  int 0x10  ; 做事

  ; 虽然说实话我更喜欢逐个字母打印...更美观。此外，你可以使用中断0x15在每个字母之间休眠一点，使其看起来很酷...

  ; 在良心下离开这里之前，我还有一件事要告诉你：堆栈
  ; 让我们快速做一个打印数字的函数（并跳过它到使用它的代码）
  jmp stack_example

; 这个函数应该打印存储在AX中的数字...事实证明，这是一项非平凡的任务
newline: 
  db 0x0d, 0x0a
print_buf:
  resb 5
  db ' ', 0
print_rax:
  lea si, [print_buf + 4]       ; 我们的策略是使用除以10的数学从右到左构建数字字符串，所以我们到达缓冲区的末尾
  ; 我想还值得一提的是这个神奇的`lea`操作码...有点像巫术。
  ; 在`[...]`里面你最多可以有`[reg_1+reg_2*const_1*const_2]`
    ; 其中`reg_1`称为`base`，因为它是基本寄存器。reg_1是另一个寄存器，乘以任意常量。
    ; 但是常量1（乘法1）和常量2必须分别是1, 2, 4或8。任何其他常量是非法的。
  mov byte [si], 0
print_digit_loop:
  xor dx, dx
  div word [10]                 ; 现在这个魔法的定义来了...你会认为div把ax除以10，把结果存到ax和dx中
  add dl, '0'                    ; 剩余部分实际上是dx，而ax是结果。类似于C中的`a = b / 10;` 和 `d = b % 10;`
  dec si
  mov [si], dl
  test ax, ax
  jnz print_digit_loop
  call print_string
  call print_newline
  ret

print_newline:
  mov si, newline
  call print_string
  ret

.stack_example:
  ; 栈通常用于参数传递...
  ; 它在操作上相当有限
  mov cx, 12345
  call print_rax  ; 12345
  push cx         ; 将要打印的下一个数字压入栈中
  xor cx, cx      ; cx = 0
  call print_rax  ; 0
  pop cx          ; 将保存的数字弹出堆栈
  call print_rax  ; 12345

  ; 但是，一些高级语言实际上使用堆栈来传递参数。这是因为调用者负责传递参数，因此可以有可变数量的参数，而不需要更多寄存器。
  ; 但是，这样一来，编译器可以优化代码，将参数传递给被调用者。

exit:
  hlt  ; 这个和第一个 hlt 是一样的
    ; 由于这是一个例子，我们将在这里停止。这是对基础知识的快速浏览，并且忽略了大量的细节。
  ; 事实上，在我们实际创建一个启动磁盘之前，我们不能运行这个代码...我们将需要处理磁盘中断和MBR中的很多东西。
  ; 但是希望这能让你了解汇编语言是如何工作的，并为你学习更多提供一个起点。

  ; 我希望你学到了很多！
  ; 如果有任何问题，请随时提出。
  ; 否则，我建议你继续深入研究，试着自己动手做一些小项目。

  ; 再见！

  ; 正如之前提到的，汇编代码结束的标记是最后一行的 `hlt` 指令。
  hlt
